# Security Policy

## Supported Versions

This project is under active development. The latest version on the `main` branch is currently supported.

| Version        | Supported |
| -------------- | --------- |
| Latest (main)  | ✅         |
| Older versions | ❌         |

---

## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

### How to report:

* Email: **[EMAIL_ADDRESS](mailto:kathurkaran077@gmail.com)**
* Or open a **private issue** (do NOT disclose sensitive details publicly)

### What to include:

* Description of the vulnerability
* Steps to reproduce
* Potential impact
* Suggested fix (if available)

---

## Response Policy

* Initial response within **48–72 hours**
* Valid issues will be investigated and patched promptly
* Critical vulnerabilities will be prioritized

---

## Guidelines

* Do NOT exploit vulnerabilities for malicious purposes
* Do NOT publicly disclose issues before they are resolved
* Responsible disclosure is expected

---

## Scope

This policy applies to:

* Application code
* Backend systems (future)
* APIs (future)
* Data processing pipelines

---

## Future Security Enhancements

As this project evolves into a full SaaS platform, the following will be implemented:

* Authentication & authorization
* Secure API access
* Rate limiting
* Data validation & sanitization
* Monitoring & logging

---

Thank you for helping improve the security of this project.
